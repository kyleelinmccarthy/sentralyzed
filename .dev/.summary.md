## What Is This?

Sentral is a collaborative workspace platform for teams. It provides project management, task tracking, communication (chat, forums), file sharing, real-time whiteboards, expense tracking, and more — all in one place.

## Stack Summary

- **Frontend**: Next.js 15 (App Router, React 19, Turbopack) + Tailwind CSS 4.0 + Zustand
- **Backend**: Hono.js REST API on Node.js + Drizzle ORM + PostgreSQL 16
- **Real-time**: WebSocket (ws) + Yjs CRDTs for collaborative editing
- **Cache/Sessions**: Redis 7
- **Validation**: Zod (shared between client and server)
- **Monorepo**: npm workspaces + Turborepo
- **Testing**: Vitest (unit) + Playwright (E2E)

## Architecture at a Glance

```
apps/api    → Hono.js REST + WebSocket (ports 3001, 3002)
apps/web    → Next.js 15 App Router (port 3000)
packages/shared → Types + Zod validators
packages/config → Shared tsconfig, eslint, prettier
```

**Key pattern**: Routes (`src/routes/`) → Services (`src/services/`) → Drizzle ORM → PostgreSQL

## Active Decisions

- **DEC-001**: .dev/ framework for project intelligence
- **DEC-002**: Hono.js over Express (performance, TS-native)
- **DEC-003**: Drizzle over Prisma (no codegen, TS-native schemas)
- **DEC-004**: Cookie sessions over JWT (simpler revocation, httpOnly)
- **DEC-005**: Railway Pro over Vercel for hosting
- **DEC-006**: Railway Postgres over Neon for database
- **DEC-007**: tsup (esbuild) over tsc for API build (ESM import rewriting + sidesteps L-001 OOM)

## Critical Learnings

- **L-001**: Better Auth generics OOM `tsc`. Type-check uses `tsc --noEmit`; build uses tsup (DEC-007) which skips type-checking entirely. Keep Better Auth imports isolated in `lib/better-auth.js` + `lib/better-auth.d.ts`.
- **L-002**: Any Dockerfile consuming `@sentral/shared` must run `npm run build --workspace=@sentral/shared` first — the `exports` field resolves to gitignored `dist/`.
- **L-003**: API build must rewrite ESM import specifiers to include `.js`. That's what tsup does. Never revert the API build to plain `tsc` — Node ESM rejects extensionless imports in production. New `packages/` consumed at runtime must do the same.
